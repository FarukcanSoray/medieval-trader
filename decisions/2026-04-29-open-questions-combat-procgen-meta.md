# Open questions resolved — combat abstracted, procgen full, meta-progression single save

## Decision
Three open design areas were resolved at intake:

- **Combat: abstracted.** Not a real-time or tactics layer. Encounters resolve as economic outcomes.
- **Procgen: full.** The map is fully procedurally generated.
- **Meta-progression: single save.** One persistent trader per world, permadeath, no cross-save meta.

## Reasoning
These constrain future system design without requiring full detailed specs yet. The CLAUDE.md left these explicitly open for the Director, and the user resolved all three in one intake answer.

## Alternatives considered
- Combat: tactics layer (rejected), real-time (rejected), abstracted (chosen).
- Procgen: none/hand-authored (rejected), full (chosen), partial (rejected).
- Meta-progression: cross-save unlocks (rejected), multiple traders (rejected), single save (chosen).

## Confidence
High. Explicit user choices on all three.
